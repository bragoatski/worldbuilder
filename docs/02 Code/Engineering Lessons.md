# Engineering Lessons - Worldbuilder

Durable, reusable gotchas specific to Worldbuilder. Format: rule + why + Verified date. Universal (cross-project) lessons live in the global layer (`~/.claude/Engineering Lessons.md`), not here. When a lesson stops being true, move it to Archive rather than deleting it.

## Reproducibility
- **Two seeded RNG streams: `sRng` = world GENERATION, `eRng` = world DYNAMICS. Both are `mulberry32`, both seeded in `initWorld` from `_seed` (eRng on a distinct offset, `_seed ^ 0x9E3779B9`), so a seed now reproduces the same TERRAIN and the same ECOLOGY run.** Generation (terrain passes, sunlight blobs, world meta, rivers, anomaly INIT) draws from `sRng` / the `sRandn`/`sTruncNorm`/`sBeta`/`sGamma` helpers. Per-tick dynamics (`floraStep`, `faunaStep`, `beachStep`, flora/fauna make+mutate+spawn, `randn()`, anomaly DRIFT in `updateAnomalyBlobs`) draw from `eRng`. The two streams are independent on purpose: ecology seeding left the generation stream byte-identical (the terrain determinism test is unchanged). The ONLY surviving raw `Math.random()` is the random-seed PICKER in `initWorld` (entropy when no seed is given) - that one must stay raw. Verified by the `is deterministic for a fixed seed (ecology)` test (two identical-seed runs -> identical flora/fauna counts). [Verified: 2026-06-22]
  - Mechanics note: the conversion was a global `Math.random()` -> `eRng()` replace plus reverting the seed picker; `randn()` is ecology-only (generation uses `sRandn`), so routing it through `eRng` does not touch generation. `truncatedNormal`/`betaapprox`/`gammaSample` are DEAD (defined, no callers - superseded by the `s*` seeded versions); they were swept too but it is inert.
- **A THIRD seeded stream `cRng` = COSMETIC genes only (the heritable size gene), seeded in `initWorld`/`restoreState` from `_seed ^ 0x85EBCA6B` via `cRandn()`.** Why a separate stream and not just another `eRng` draw: ANY draw added to `eRng` shifts the whole downstream ecology phase (the RNG-reshuffle gotcha in Ecosystem tuning), so a "cosmetic-first, balance-safe" gene that drew from `eRng` would silently change the hard-won C2 trajectory even though the gene never touches energy/eat. Keeping size-gene drift on `cRng` leaves `eRng` byte-identical, so the ecology run is PROVABLY unchanged - confirmed by `measure --seeds=8` before/after the size gene returning identical numbers (0% extinction, carnivore-persistence 75% 6/8, final fauna 73.6, flora 2263.8, cap-hits 0). The rule for any future VISUAL/observational gene: (a) it must never be read by `faunaStep`/`scoreTileForFauna`/the energy/repro fields, and (b) it draws only from `cRng` - then it needs NO harness re-tune. The moment a gene affects behavior it moves onto `eRng` and goes through the measure->A/B loop like any rate change. [Verified: 2026-06-30]

## Headless testing & the harness (added 2026-06-22)
- **The sim runs headless in Node now; that is what makes the gate automated and the harness possible.** `src/main.js` exports pure entry points (`initWorld`, `runAssertions`, `step`, `seedFloraCluster`, `seedFaunaGroup`, `landCoverage`, live `flora`/`fauna`/`tick`). `runTests` was split into a pure `runAssertions()` (returns `{out,pass,fail}`) plus a DOM wrapper; `init` into a DOM-free `initWorld(seed)` plus a DOM wrapper. The in-page `T` test now also runs in vitest (`src/sim.test.js`). [Verified: 2026-06-22]
- **`main.js` is still the browser ENTRY, so importing it in Node runs its UI wiring. A permissive Proxy DOM stub (`scripts/headless-dom.mjs`) lets that wiring no-op so tests/harness can import it.** This stub is interim scaffolding; it goes away once the sim core is split out of the shell into its own DOM-free module (the file split was deliberately deferred - headless already works via the stub, so the split is now cleanliness, not capability). [Verified: 2026-06-22]
- **Measurement harness: `npm run measure` (`scripts/harness.mjs`).** Runs N seeds x M ticks headless and reports extinction rate, time-to-extinction, final/oscillating populations, variance, PLUS cycle-aware metrics (predator-prey phase lag, per-trophic period/amplitude, completed cycles, carnivore-persistence, per-trophic min floor, cap-hit count, herd spatial dispersion). Flags: `--seeds --warmup --ticks --herb --carn --flora --sample --repeat --snapshot --traj`, plus climate A/B flags `--seasons --anomalies --volcano --seasonlen --intensity` (warmup always runs climate-off, the toggles apply only to the measured window, so off-vs-on is a clean same-warmed-world compare). This is the instrument for tuning ecosystem balance by numbers instead of eyeballing one animation. [Verified: 2026-06-22; climate flags 2026-06-29]
- **Phase lag is the headline tuning metric, but only trust it where the coupling is real.** The harness cross-correlates herb(t) vs carn(t); a positive lag means carnivores peak AFTER prey (a real cycle). On a flat/degenerate run the lag search returns a meaningless extreme at correlation ~0, so the per-seed line prints the correlation `r` and the AGGREGATE phase lag averages only seeds with `r > 0.3`. Read the per-seed `r` before believing a lag. [Verified: 2026-06-22]

## Sim behavior (added 2026-06-22)
- **Terrain genesis is SLOW: ~3,000-4,000 ticks to reach ~30% land (a few hundred ticks gives ~1%).** Confirmed in-browser (tick 509 -> "Land 0.4%") and via the harness. Consequence: any headless ecology study must warm up several thousand ticks before seeding fauna, or it measures a near-empty ocean where flora can't establish and fauna starve. Harness defaults warm up 3,000 ticks for this reason. The snapshot path below now amortizes this cost across replays. [Verified: 2026-06-22]
- **Balance lever surfaced by the test gate: `herbivoreEatSpeed === herbivoreSpeed` (both 20).** The in-page assertion "Herb eats slower than moves" encoded an intended overgrazing-prevention invariant (eat cooldown > move cooldown) that the config violates. Relaxed to `>=` to keep behavior unchanged and the gate honest (see the `// ponytail:` note at the assertion). Making eat slower than move is a real, untested ecosystem-balance change to evaluate WITH the harness, not blindly. [Verified: 2026-06-22]

## Snapshot / restore (added 2026-06-22)
- **`snapshotState()` / `restoreState(snap)` (exported from `main.js`) warm the slow terrain ONCE per seed and replay the ecology window many times (`--snapshot`).** Implementation: build one object holding every piece of resumable state (terrain + climate + volcano typed arrays, flora/fauna/remnant lists, world meta, scalars) and `structuredClone` it - that deep-copies in one call AND preserves each typed array's view type (Uint8Array stays Uint8Array). `restoreState` clones again before assigning, so ONE snapshot can be restored repeatedly without aliasing. The module vars are reassigned (they are `var` at module scope; the exported `flora`/`fauna`/`tick` are live bindings, so importers see the new values). [Verified: 2026-06-22]
- **mulberry32's internal counter is not externally readable, so the snapshot stores the SEED and `restoreState` RE-SEEDS both streams (`sRng=mulberry32(_seed)`, `eRng=mulberry32((_seed^0x9E3779B9)>>>0)`) exactly as `initWorld` does.** Consequence to remember: a restored run re-seeds the RNG to its tick-0 state, so fauna are seeded at a DIFFERENT RNG phase than a continuous (non-snapshot) run that warmed eRng for 3k ticks first. Snapshot numbers are therefore NOT byte-equal to non-snapshot numbers - they are their own internally-consistent baseline. Two restores of one snapshot DO match (the determinism-through-snapshot test proves it), so A/B tuning within snapshot mode (same snapshot, only CFG differs) is clean. If you switch the tuning loop to `--snapshot`, re-baseline under snapshot mode first. [Verified: 2026-06-22]

## Dev-system gotcha (added 2026-06-22)
- **Taipan's `precommit-gate.mjs` hook runs TAIPAN's `npm run typecheck` for ANY `git commit` in the session - including commits in the sibling worldbuilder repo - and DENIES the whole Bash call if Taipan is red.** So a worldbuilder commit can be blocked by an unrelated mid-edit red state in Taipan (e.g. Kevin editing Taipan in parallel). Two practical consequences: (1) the hook blocks the ENTIRE Bash command, so a `git add ... && git commit` that's denied also skips the `git add` - stage in a separate call first; (2) if blocked, check Taipan's gate is green (it may just be transient) and retry, rather than editing Taipan. [Verified: 2026-06-22]

## Ecosystem tuning (added 2026-06-22)
- **The world is NON-STATIONARY: terrain genesis never stops, so total flora (the food base / carrying capacity) keeps rising for thousands of ticks.** Measured: warmup 3000 -> ~24% land / ~2,000 flora; +3000 more ticks -> ~54% land / ~11,000 flora. Consequence for tuning: the standard `measure --ticks=1000` protocol judges the ecology against a near-static SNAPSHOT (land ~24%), NOT a steady state. A config that balances there (0 extinction, healthy +lag cycle at 1000t) can still hit the population cap (herbivores boom with 5x flora) and crash seeds over a 3000t horizon. So "window-balanced" != "steady-state balanced"; long-run ecology studies must either warm up to a land plateau or define balance per development stage. [Verified: 2026-06-22]
- **RNG-stream reshuffle: any knob whose code path GATES a conditional `eRng()` call changes the per-tick RNG draw COUNT, which shifts the entire downstream stream - so its A/B is NOT a same-world comparison.** The carnivore-rescue branch (`cc<cap && hc>=minPrey && eRng()<rate*hc`) draws eRng only when the first conditions hold; changing minPrey/rate/cap therefore reshuffles which seed does what (seed trajectories swung 62C->4C purely from reshuffle), making per-seed reasoning unreliable and adding noise even to aggregates. Knobs that change only a magnitude with NO eRng gate (carnivoreEatGain; the herbivoreCrowding scoring weight) give clean same-world A/Bs. Rule: prefer non-eRng-gating knobs for clean attribution; for eRng-gating knobs, trust only large aggregate moves over many seeds, not per-seed deltas. [Verified: 2026-06-22]
- **`carnivoreEatGain` tunes the predator NUMERICAL response (how fast predator NUMBERS grow) + starvation, NOT the per-capita KILL RATE.** Predators kill whenever `carnivoreEatSpeed` (the eatCD cooldown) is ready and prey are present, regardless of energy-per-kill. So LOWERING eatGain to "gentle the predators" backfires: same kill rate, less energy banked, hungrier predators need MORE kills -> more predation pressure, not less (eatGain 40->32 raised extinction). The kill-RATE / predation-pressure lever is `carnivoreEatSpeed` (lengthen the cooldown) - but even that was a no-op when one kill (eatGain 40) nearly fills the tank (maxEnergy 120), so the cooldown was not binding. Amplitude in this regime is driven by predator (large-eatGain, cheap-repro) ECONOMICS, not kill rate. [Verified: 2026-06-22]
- **Conspecific crowding (dispersion) is the LOAD-BEARING lever for predator-prey persistence - rate tuning (predator lag, spawn rescue) cannot substitute for it.** Adding a herbivore "avoid tiles dense with other herbivores" term to `scoreTileForFauna` (knob C) flipped the system at weight 2.0 from 0% carnivore-persistence / 17% extinction to 83% persistence / 0% extinction / +68t phase lag. Why: it fragments the single moving herd into out-of-phase patches, so a local crash leaves neighbors to recolonize instead of a global synchronous collapse (the proposal's reason-2). Predator-lag (knob A) and spawn-rescue (knob D) tuned the rates but left fragile stragglers until dispersion broke the synchrony. [Verified: 2026-06-22]
- **Long-run ecology is limited by the PARADOX OF ENRICHMENT, made worse by the non-stationary world: no fixed rate-knob set gives steady-state balance while the carrying capacity K keeps rising.** As terrain/flora grow (K rises), the predator-prey limit-cycle AMPLITUDE grows until troughs hit the absorbing-zero wall and seeds go extinct (Rosenzweig 1971). Measured: C2 is balanced at 1000t/land~24% (0% extinction) but at 3000t/land~54% amplitude diverges and 33% of seeds crash. Both prey-side rate fixes FAIL for the same root reason - local density-dependent birth is inert (knob C already keeps local density low), and MORE dispersion (crowding 2.0->2.5) BACKFIRES because it raises effective K (herbivores colonize more of the map -> bigger booms; cap-hits 27->388). The destabilizer is K itself, not the rates. Real fixes are beyond rate tuning: bound K (cap terrain/flora growth, generation-side), or add an enrichment-robust mechanism (predator interference / prey predation-refuge). For a living-world sim, accepting the organic boom/bust as K rises is also legitimate. [Verified: 2026-06-23]

## Fauna distribution vs the balance (added 2026-06-23)
- **DESYNCING the herbivore herd (to kill the synchronized "pulsing ring" look) STARVES the carnivores: the clustered prey ring IS the predators' food supply.** Giving herbivores a per-individual move/eat PHASE (so they stop marching + reproducing in lockstep) disperses them; carnivores hunt by local prey DENSITY (`scoreTileForFauna` sums herbivores in nearby tiles + a scent ring), so dispersed prey collapses carnivore-persistence 83% -> 0% (all 6 seeds), extinction 0% -> 33%. This is a real dynamical effect, not RNG reshuffle - it was measured with a POSITION-based phase (`moveCD=(x*7+y*13)%speed`, `eatCD=(x*11+y*5)%eatSpeed`) that draws NO `eRng`, so the stream is unchanged and the A/B is clean. Compensate by helping predators exploit dispersed prey: `carnivoreEatGain 40->55`, scent range ring 2-3 -> 2-5 (weaker tail weights), stronger knob-D rescue (`carnivoreRescueRate 0.00005->0.0001`, `minPrey 30->20`, `carnCap 4->6`). Result (10 seeds): 0% extinction, 80% carnivore-persistence = C2 (0%/83%). Desyncing ONLY herbivores (carnivores keep `moveCD:0` full cadence) is also required - desyncing carnivores too left them at 0%. [Verified: 2026-06-25]
- **Fauna PERSISTENCE is too noisy to tune on 6 seeds; use >=10.** The SAME code (the desync+boost above) read 67% persistence on the default 6 seeds and 80% on 10 - the gap was 2 chaotic seeds, not a real difference. A weaker boost variant even read 0% extinction on 6 seeds one run and 33% the next region of the noise. Extinction/persistence are near-absorbing-boundary outcomes with high seed variance; trust the >=10-seed aggregate, not a 6-seed point. [Verified: 2026-06-25]
- **Changing fauna SCORING (`scoreTileForFauna`) to reshape WHERE animals live - even "distribution only" (harsh-biome avoidance, water-shore attraction) - regresses the C2 predator-prey balance and must be treated as its own harness-tuned task, not a quick add.** Measured on the standard protocol: a desert/arctic harshness multiplier on `computeFaunaClimateFit` (survival) gave 33% extinction / 50% carnivore-persistence / phase-lag -92t; moving it to movement-only scoring still gave 17% / 50% / -124t (C2 baseline is 0% / 83% / +68t). Why: the foraging score is what knob C (`herbivoreCrowding`) uses to FRAGMENT the herd into out-of-phase patches; adding a strong biome-avoid penalty (weight 4) or a water-attractor re-concentrates the herd and undoes the dispersion that keeps the cycle off the absorbing-zero wall. Render-only suppression (not DRAWING fauna/flora on lake tiles in `draw`) is balance-neutral and fine. Lesson: any "fauna should be rarer / crowd water / avoid deserts" feel change needs the measure->A/B->keep-if-metrics-improve loop, like the C2 work. [Verified: 2026-06-23]

## Rivers / hydrology (added 2026-06-23)
- **Rivers are generated by a standard hydrology pipeline, NOT a tracer: priority-flood depression fill (Barnes 2014) -> D8 flow receivers -> flow accumulation -> threshold.** The old `generateRivers` was a greedy downhill trace that dead-ended in local minima ("basin lakes") and was capped at ~7 rivers, so coherent dendritic networks never formed. The pipeline guarantees, BY CONSTRUCTION, that every land cell drains to the sea (no orphaned stubs) and that rivers are dendritic, with width from drainage. Combined fill+flow-dirs are done in one priority-flood pass (a binary min-heap seeded from every ocean cell + the map border); the first cell to reach a neighbour becomes its receiver, which also resolves flats. Structural invariants are gate-tested on a synthetic fractal continent in `sim.test.js` (every river reaches sea/lake/edge; no dead-ends; dendritic not blobby). [Verified: 2026-06-23]
- **This terrain CAPS river size: max drainage area is ~27-44 cells at typical land coverage (~55%), ~345 on a near-complete continent (~89%).** The default world is low-relief with a crenellated coast and interior seas, so every point is within ~30 cells of water and catchments stay tiny - big trunk rivers cannot form until the land consolidates. Lever: raise `maxLandCap` (made the default 0.90, was 0.60) so the world grows into a continent; rivers then reach maxAcc ~345 and read as real river systems. Measure maxAcc on the actual warmed terrain before picking the river threshold (the threshold->coverage curve is the tuning instrument). [Verified: 2026-06-23]
  - **2026-06-24:** `RIVER_ACCUM_THRESHOLD` moved into `CFG.riverAccumThreshold` (default lowered 22 -> 14) and exposed as a live "River Density" slider (Terrain panel, inverted: drag right = denser; regenerates rivers on input). Measured threshold->river-coverage at seed 1000 / 88.7% land: th 8 -> 9.2%, 12 -> 6.6%, **14 -> 6.0%**, 18 -> 5.1%, 22 -> 4.1% (old), 30 -> 3.2%. 14 reads as "clearly present, still dendritic" and also makes rivers appear at lower land coverage (more catchments cross a lower bar) - the old 22 looked sparse/empty on under-developed (~56% land) worlds. Rivers were otherwise UNCHANGED by the flora work.
- **Natural fill-lakes only form in LOW ground (coastal), never at high river sources.** Kevin wanted lakes at the river STARTS, so an elevation filter on natural lakes leaves ~none, and the headwater lakes are SYNTHESIZED: `generateRivers` floods a few big, well-spaced disks at the highest river heads (`SOURCE_LAKE_*`), and each river springs from / overflows out of one. River WIDTH is uniform per river (set by the river's LENGTH via a main-stem decomposition using the longest-upstream-length `ul`), not growing downstream - that was an explicit design call. [Verified: 2026-06-23]
- **`maxLandCap = 0.90` default means the DEFAULT ecology now runs at ~90% land, far above the ~24% the shipped balance (C2) was tuned at.** By the paradox-of-enrichment lesson this tends boomier/less-balanced; it is a known tradeoff Kevin accepted for the river/terrain look (the slider dials it back). If default ecology balance regresses, re-tune at high land or lower the default cap. [Verified: 2026-06-23]
- **2026-06-25 river-feel pass (all render/generation, ecology untouched):** lakes + rivers now share one color (`LAKE_COLOR = RIVER_COLOR = #3aa6e0`) so overlaps read as one body; the per-headwater tiny source pool was removed (looked bad); source lakes cut 6 -> 3 and radius ~halved (`SOURCE_LAKE_*`); **never bleed into the sea** - a source lake is rejected if its blob footprint touches ocean/map-edge, river MOUTHS end inside their land cell with a flat (`butt`) cap + straight pulled-back final segment, and the braided delta was removed (it was a bleed source); routing smoothing `RIVER_SMOOTH_PASSES 3 -> 6` to merge the parallel rivulets into fewer, longer continent-stretching channels. Still gate-blind - shipped to live for visual review.
- **2026-06-28: "no rivers" in a normal sim is a LAND-LEVEL problem, not a bug. Rivers only form at high land; the fix is a lower default threshold, NOT auto-generation.** `scripts/river-diag.mjs` (warm headless, sweep `riverAccumThreshold`, report coverage + a blob-ratio = solid 2x2 river fraction) showed at the old default 14: 0 rivers below ~40% land, 0.5% at 56% land, the good networks only at ~88% land (~10k ticks). Lowered the default `riverAccumThreshold 14 -> 6` (slider dense-end `min 6 -> 3`): ~2.1% coverage at 56% land, ~9.4% at high land, and on the REAL low-relief terrain it stays dendritic (blob <12%) across the range. NOTE the gameplay default is denser than the structural unit test tolerates, because the test's synthetic continent is a high-relief cone that is far more blob-prone (40% blob at th6) than real terrain - so the test PINS its own threshold (14) and validates pipeline SHAPE, not the gameplay density. Also: `RIVER_SMOOTH_PASSES` 3 vs 6 barely changes coverage (4.8% vs 4.6% at high land) - not a useful lever. [Verified: 2026-06-28]
- **Auto-generating rivers in `step()` (so they appear without the button) costs predator balance and was REVERTED: rivers feed `waterDist`/aridity, which concentrates flora, which shifts the predator-prey dynamics.** Measured (10 seeds): adding `if(tick%300===0)generateRivers()` to the step loop dropped carnivore-persistence 80% -> 60% (extinction stayed 0%); specific seeds flipped (a carnivore-boom seed lost its predators). Rivers were historically ABSENT from the headless harness, so any change that makes them run per-tick re-enters them into the measured balance. Kept rivers MANUAL (button + River Density slider) so the harness stays river-free and the 0%/80% balance is byte-untouched. If auto-rivers are ever wanted, treat the river-aridity coupling as a balance change and re-tune. [Verified: 2026-06-28]
- **The render is GATE-BLIND, so rivers are verified visually via a `?debug` handle, not the gate.** `scripts/make-preview-world.mjs` warms terrain headless and serializes a loadable snapshot (the `buildSnapshot` export, `wb-eco-1` format - now also restores `grid`, which `importJSON` previously dropped, so loaded worlds were all-ocean); the app exposes `window.__wb` ONLY with `?debug` in the URL, letting Playwright load a developed world + generate rivers + screenshot without the slow in-browser warmup or a file dialog. Preview JSONs are gitignored (large). This is the reusable path for any future gate-blind terrain/river verification. [Verified: 2026-06-23]

## Flora distribution / clustering (added 2026-06-24)
- **"Flora overruns everything" is a HIGH-LAND phenomenon; the standard `measure` protocol (warmup 3000 -> ~24% land) cannot see it.** At ~24% land the world is uniformly wet - deserts are ~0% of land and ~72% of land is within 1 tile of water - so flora-distribution levers (water-clustering, desert-avoidance) are essentially INERT there. The overrun appears only as the world matures toward the default `maxLandCap=0.90` (warmup 6000 -> ~76% land, flora covers ~66% of land, fauna pegged at the cap). RULE: measure flora DISTRIBUTION + coverage at high land (warmup >=6000); measure BALANCE at the standard 24% protocol (where C2 is defined). [Verified: 2026-06-24]
- **Aridity is too FLAT a signal to make flora "cluster on rivers/lakes".** Aridity is coast-distance-dominated (`aridityDistK`), the river effect is small (`RIVER_ARIDITY_EFFECT=0.6`), and the mean stays ~2.75 even at 76% land - so weighting flora by aridity barely moves the distribution. For water-clustering use an EXPLICIT `waterDist` field (BFS distance to ocean/coast + river/lake tiles, `computeWaterDist`, refreshed with `computeAridity`/on restore). Aridity is still the right signal for the desert-avoidance brake. NOTE: rivers are not generated in the headless harness, so `waterDist` is coast-only there; river/lake hugging is GATE-BLIND (verify in-browser on a loaded preview + generated rivers). [Verified: 2026-06-24]
- **Placement bias alone barely changes flora distribution - the dominant dynamics are local SPREAD + ADAPTATION, so you must brake SURVIVAL (health), not just where seeds land.** Flora health is scored against the plant's OWN `prefArid` (mutation pulls prefs toward local conditions), so dry-evolved variants thrive in dry interior; a placement-only water/desert weight (even strong) leaves established flora untouched (measured: water aridity moved 2.99 -> 2.81 at most). The levers that actually relocate flora are an ABSOLUTE moisture/water brake folded into `computeFloraHealth` (slows spread via health^2, speeds death) - that is what clears deserts (share 1.6% -> 0%) and pulls flora onto water. [Verified: 2026-06-24]
- **A global flora reduction (spread/perTile/spawn) FIXES high-land overrun but STARVES the low-land C2 balance (extinction 25% -> 75%). The fix is LAND-ADAPTIVE maturity-thinning, not a flat cut.** `floraLandThin` scales flora spread + spawn DOWN as land coverage rises, with ZERO effect below `floraLandThinStart` (0.4). So early small-island worlds stay lush (C2 balance, tuned at ~24% land, is byte-untouched) while a matured ~90%-land continent stops being carpeted (coverage 66% -> 41%, flora ~halved, cycle still 0% extinction / carnivores persist). This is the general pattern for tuning a NON-STATIONARY world: condition the knob on the development stage, do not cut globally. [Verified: 2026-06-24]
- **Certify a placement-ALGORITHM change with a clean same-code A/B, never against the old-code baseline.** Switching flora placement from a guard-loop to `pickFloraTile` (a FIXED-count weighted sample - K candidates, proportional pick, so tuning a knob does NOT change the eRng draw count) is the right design (rejection sampling reshuffles the stream - see the RNG-reshuffle lesson). BUT the new algorithm draws a different NUMBER of eRng values than the old guard-loop, so `new-code-shipped vs original-C2-baseline` is reshuffled, not same-world: it read a scary 17%/50% while flora was provably IDENTICAL. The valid certification is `flora-ab` OFF-vs-ON (both new code, draw-stable, same seeds), which showed neutral-to-better (8 seeds: extinction 13% vs 25%). [Verified: 2026-06-24]
- **Measuring flora DISTRIBUTION requires warming WITH the variant CFG; the shared-snapshot A/B only works for BALANCE.** Distribution is established over the whole run - a 1500t window from a uniform snapshot cannot redistribute the ~18k already-placed plants (placement biases only NEW flora; spread is local). `scripts/flora-ab.mjs` warms each variant separately for this reason (terrain depends only on the seed/`sRng`, so it is still a fair same-terrain compare); balance metrics, which re-seed fauna fresh, would be fine on a shared snapshot. The harness now reports a flora-distribution block (coverage, aridity vs land, desert share, near-water share) as the instrument for this work. [Verified: 2026-06-24]

## Climate / seasons (added 2026-06-29)
- **Climate is an OFFSET on a cached genesis baseline, not an integrated delta on the live field.** `computeTemperature`/`computeAridity` write `baseTemp`/`baseArid` (climate-independent, a function of sunlight/elev/coast-distance); `applyClimate` runs EVERY tick and writes the live `tempField`/`aridity = base + seasonal + anomaly + volcano offsets`. Each forcing is recomputed from scratch (never `field += delta`), so toggling any of them just adds/removes a bounded swing and NOTHING accumulates. When all climate toggles are off, `applyClimate` copies base verbatim, so the seasons-off ecology is byte-identical to the pre-2026-06-29 baseline (the ecology-determinism + snapshot tests still pass) - that is the property that let the redesign ship without re-tuning C2. [Verified: 2026-06-29]
- **ROOT CAUSE of the old broken "seasons" (pre-2026-06-29), for anyone touching climate again: (1) the seasonal delta was INTEGRATED onto `tempField` every tick, and (2) `computeTemperature`/`computeAridity` (which OVERWRITE the field) were SUPPRESSED whenever any climate toggle was on, running only on genesis events.** Two failures fell out: a DRIFT (the plateau waveform averaged ~-0.15, not 0, so integrating it marched the world permanently colder + drier) and REGIME-DEPENDENCE (while land was still growing, frequent genesis reset the field every tick, so seasons were nearly INVISIBLE; only once land hit `maxLandCap` and genesis stopped did the delta accumulate - then as drift). Probed directly with `scripts/season-probe.mjs`: matured world, old model, phase-0 `Tmean` drifted 4.37->3.97 over 3000t; new model holds 4.31 every year (off-control also 4.31). Lesson: a periodic forcing must be a ZERO-MEAN offset on a maintained baseline, never an integral, and the baseline must refresh independently of the forcing's on/off state. [Verified: 2026-06-29]
- **`seasonWave(phase)` is a SYMMETRIC trapezoid in [-1,1] (warm plateau ~.25, cold plateau ~.75); symmetry guarantees zero mean = no climate drift, gate-tested ("Season wave zero-mean").** Amplitude lives in CFG knobs (`seasonalTempAmp` 1.5, `seasonalAridAmp` 0.9 with the warm-season-is-moister sign kept from the old model; `anomaly*`/`volcano*` variants), scaled per-tile by elevation attenuation and globally by `climateIntensity` (ice age 1.8, volcanic 1.5). Magnitude is INDEPENDENT of `climateSeasonLength` (length only sets cycle speed), so a short season in the harness is a faithful magnitude test of the default long season. [Verified: 2026-06-29]
- **Seasons at the moderate ±1.5 amplitude are BALANCE-SAFE (neutral-to-slightly-stabilizing), measured seasons-off vs seasons-on at matched protocols (harness `--seasons` flag).** Clean C2 window (8 seeds, 1000t, one full season): cap-hits 0 both, survivor fauna 64->69, fauna oscillation 85->69 (seasons slightly DAMP the boom-bust swing). Harsh long window (6 seeds, 4000t, in the accepted paradox-of-enrichment regime): extinction 33%->17%, carnivore-persistence 67%->83% - seasons do not destabilize even there. Why seasons don't break the herd-fragmentation balance (unlike the reverted fauna-scoring changes): seasons shift the climate FIELDS uniformly, they don't touch `scoreTileForFauna`, so the dispersion that keeps the predator-prey cycle off the absorbing-zero wall is untouched. [Verified: 2026-06-29]

## God powers / interventions (added 2026-06-30)
- **A user INTERVENTION (god-power) is balance-safe BY CONSTRUCTION when it never runs inside `step()`: the harness + tests only ever exercise `step()`, so nothing an intervention does can advance the MEASURED eRng stream.** Chunk 3's land brush + meteor/drought/bloom are pure sim-core mutations called ONLY from UI hooks (button click / canvas click), never from the per-tick pipeline (`step`/`floraStep`/`faunaStep`/`scoreTileForFauna` untouched). So even though they are behavior-touching (they change terrain/flora/fauna), the harness runs them zero times and the C2 balance is byte-identical - `measure --seeds=8` before vs after returned identical numbers (0% extinction, carn-persistence 75% 6/8, final fauna 73.6, flora 2263.8, cap-hits 0), the same proof shape as the chunk-2 cosmetic genes. The rule: an intervention only needs the harness measure->A/B loop if it wires itself INTO the step path (as auto-rivers did, and were reverted for exactly this reason). Draw randomness from `eRng` (keeps `main.js` free of stray `Math.random`); it advances the stream only at the arbitrary tick the human acts, which is the intended intervention, not a balance regression. [Verified: 2026-06-30]
- **An intervention's LASTING consequence must live on state the per-tick pipeline does NOT overwrite. A direct `aridity[]` bump would be WASHED OUT within ~20 ticks** because `applyClimate` rewrites the live field every tick as `base + offsets` and `computeTemperature`/`computeAridity` refresh the base on genesis / every 20th tick (the 2026-06-29 climate-offset redesign). So "drought" is implemented as a FLORA DIEBACK (kill a fraction, weighted by tile aridity so dry ground scorches worst), not an aridity poke - the flora list persists, and the downstream cascade (fewer plants -> herbivores starve -> carnivores starve) is then narrated FOR FREE by the existing `chronicleSample` crash/extinct detectors over the following ticks. Terrain shocks (meteor crater, brush) persist naturally because `elev`/`grid` are only changed generation-side, slowly. General pattern: for a durable intervention, mutate population lists or elevation, not a field the climate/genesis loop reclaims. [Verified: 2026-06-30]
- **Reuse the null-then-compact life-array convention for bulk removal.** `flora`/`fauna` tolerate `null` holes everywhere (every iterator does `if(!f)continue`) and are compacted at the end of `floraStep`/`faunaStep` via `.filter(f=>f!==null)`. Bulk-kill helpers (`_killLifeAt`, drought) null the entries then call a local `_compactLife()` so counts + the render are immediate; killed fauna push a `deathParticles` `'kill'` marker so a meteor/sink visibly flashes. Do NOT `splice` mid-array (index-shift bugs); match the established null-then-filter idiom. [Verified: 2026-06-30]

## Shareable worlds / world code (added 2026-06-30)
- **A world's full IDENTITY is its seed + config, not its baked state - so a permalink is `{ seed, preset, cfg-diff }`, not the heavy snapshot.** Terrain + ecology are deterministic from the seeded streams (`sRng`/`eRng`/`cRng`, all re-seeded from `_seed` in `initWorld`), and `WORLD` meta is re-derived from `_seed` by `pickWorldMeta` - so it does NOT need to be encoded. Replaying `initWorld(seed)` with the same CFG reproduces the same evolving world (proven by the chunk-4 `applying a world code reproduces the same evolving world` test: apply -> step 300 -> landCoverage matches on repeat). The heavy `wb-eco-1` JSON snapshot (full grid/elev/flora/fauna arrays) stays the download path for an EXACT already-evolved state; the URL permalink is the cheap GENESIS recipe. Same balance-safe shape as chunks 2-3: `applyWorldCode` mutates the sim only via `initWorld` (the preset selector's re-genesis path), never in `step()`, so the harness is byte-identical - no A/B needed. [Verified: 2026-06-30]
- **Serialize the INDEPENDENT config, never DERIVED keys - `applyElevationIntensity` recomputes 5 CFG keys from `elevationIntensity` on every `initWorld`, so a naive diff-from-`DEFAULT_CFG` always dirties them.** `clusterSpikeRate`/`clusterPlusChance`/`mountainAdjUpliftProb`/`hillAdjUpliftProb`/`rareSurgeProb` are set at CFG-definition AND re-derived at the end of `initWorld` (to EI=1.0 values that DIFFER from their literal defaults), so after ANY init a diff-from-default lists them even for a pristine world. `buildWorldCode` excludes them (`_DERIVED_CFG_KEYS`) and ships `elevationIntensity` instead; on load they recompute, so nothing is lost and a default world encodes to an empty diff (minimal URL). General rule: when diffing a config for serialization, drop any field that is a deterministic function of another field recomputed on load. (Also: `CFG` is a persistent global that `initWorld` does NOT reset, so tests leak CFG state to each other - reset via `applyWorldCode({v:1,seed,cfg:{}})` before asserting a clean diff.) [Verified: 2026-06-30]
- **Treat a `?w=` URL as UNTRUSTED input at the boundary.** `applyWorldCode` validates the version + seed and throws on a bad code, and layers only KNOWN CFG keys (`DEFAULT_CFG.hasOwnProperty(k)`) whose value TYPE matches the default (`typeof data.cfg[ck]===typeof DEFAULT_CFG[ck]`) - so a hand-edited link cannot inject arbitrary keys or wrong-typed values (gate-tested). `init()` consumes the param ONCE via a module-load `_pendingWorldCode` one-shot, so a later preset change / reset rolls a FRESH world instead of re-restoring the link; a malformed code shows an error and falls through to a normal world. [Verified: 2026-06-30]
